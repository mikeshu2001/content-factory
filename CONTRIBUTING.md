# Как предлагать изменения

Этот репозиторий должен оставаться универсальным каркасом редакционной системы.

Подходящие изменения:

- улучшают конвейер;
- уточняют шаблоны;
- добавляют нейтральные примеры;
- добавляют универсальные антипаттерны письма;
- улучшают инструкции по установке для разных агентных инструментов.

Не стоит добавлять:

- правила конкретного бренда;
- приватные примеры;
- призыв к действию конкретной платформы в ядро системы;
- готовую редполитику одного проекта вместо шаблона;
- сгенерированные тексты из `output/`;
- реальные исходники из `input/`.

Если добавляете новое правило, по возможности добавьте пару "плохо / хорошо".
