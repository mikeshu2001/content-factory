# Content Factory

Content Factory - это каркас редакционной системы на ИИ-агентах. Он помогает собрать свою маленькую редакцию: с ролями, правилами, примерами, проверкой плана и финальной редактурой.

Это не готовая редполитика чужого проекта и не промпт "напиши статью". Репозиторий показывает, как построить собственную систему производства текстов: под свою аудиторию, свои форматы и свой голос.

[Телеграм-канал автора](https://t.me/misha_davai_po_novoi)

## Что внутри

- `agents/` - исходные инструкции рабочих ролей: researcher, article-researcher, planner, writer.
- `skills/good-writing/` - исходные универсальные правила хорошего письма и антипаттерны.
- `skills/write-*` - исходные команды-дирижеры для запуска конвейера.
- `skills/redpolicy-*` - исходные редполитики универсальных форматов.
- `templates/` - вспомогательные шаблоны для настройки своего проекта.
- `docs/` - объяснение архитектуры и перенесенный обзор исходной системы.
- `examples/` - место для эталонных текстов пользователя.
- `input/` - сюда кладут исходные материалы.
- `output/` - сюда система сохраняет план, черновик и финальный текст.

## Главная идея

Хороший текст редко получается одним запросом. Нужен процесс:

1. Собрать фактуру, если она нужна.
2. Сделать план.
3. Проверить план до написания.
4. Согласовать план с человеком.
5. Написать черновик по утвержденному плану.
6. Отредактировать текст целиком.
7. Сохранить финальную версию.

Такой процесс снижает случайность. Агент не пытается одновременно исследовать, структурировать, писать и редактировать. У каждой роли есть понятная задача.

## Что это не делает за вас

Система не знает ваш бренд, аудиторию и стиль. Их нужно описать.

Перед первым запуском заполните:

- `templates/project-profile.md`;
- `templates/content-type-profile.md`;
- `templates/redpolicy-template.md`;
- папку `examples/` своими эталонными текстами.

Без этого агент напишет средний аккуратный текст. С этим он начнет писать как редакция с понятными правилами.

## Быстрый старт

1. Откройте `QUICKSTART.md`.
2. Заполните профиль проекта.
3. Опишите один формат текста.
4. Добавьте 2-3 примера хорошего результата.
5. Положите исходник в `input/`.
6. Запустите подходящий `write-*` скилл из `skills/`.

## Какие форматы можно собрать

Любые, если для них есть правила и примеры:

- объясняющая статья;
- обзор;
- подборка;
- инструкция;
- урок;
- сценарий;
- письмо;
- описание продукта;
- история от первого лица;
- база знаний.

В репозитории нет готовой привязки к конкретному каналу, клубу, продукту или медиа. Все такие вещи должны жить в вашем `project-profile.md`.

## Основное правило настройки

Не начинайте с промпта. Начинайте с редакционной логики:

- кто читатель;
- зачем ему текст;
- что он должен понять;
- какие факты можно использовать;
- какой тон уместен;
- какие ошибки нельзя допускать;
- какие примеры считаются эталоном.

Промпт - это только способ передать эту логику агенту.
