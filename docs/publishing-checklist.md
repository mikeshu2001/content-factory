# Чек-лист перед публикацией

Перед заливкой репозитория на GitHub проверьте эти пункты.

## Репозиторий

- `input/` пустой, кроме `.gitkeep`.
- `output/` пустой, кроме `.gitkeep`.
- Локальные настройки не попали в репозиторий.
- Приватные заметки и рабочие логи не попали в репозиторий.
- Реальные материалы клиентов, учеников или участников не попали в репозиторий без разрешения.
- Лицензия выбрана.
- В README стоит правильная авторская ссылка.

## Содержимое

- В ядре не зашит конкретный бренд.
- Примеры нейтральные или явно разрешены для публикации.
- Редполитики оформлены как шаблоны, а не как скрытые правила частного проекта.
- Антипаттерны универсальны и не зависят от одной ниши.
- Документы объясняют, как построить систему, а не как скопировать чужой голос.

## Удобство

- Новый пользователь понимает процесс из `README.md`.
- Новый пользователь может начать по `QUICKSTART.md`.
- Шаблоны можно заполнить.
- Агенты знают, какие файлы читать.
- Конвейер сохраняет `research.md`, `plan.md`, `draft.md` и `final.md`.
