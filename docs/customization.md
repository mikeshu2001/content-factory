# Настройка под свой проект

Content Factory становится полезной только после адаптации под ваш проект.

## 1. Разделите универсальные правила и правила проекта

Универсальные правила лежат здесь:

- `skills/good-writing/SKILL.md`;
- `skills/good-writing/antipatterns.md`.

Правила проекта лежат в профиле и редполитиках:

- аудитория;
- тон;
- разрешенные слова;
- запрещенные слова;
- призыв к действию;
- ссылки;
- формат публикации;
- примеры.

Не меняйте универсальные правила каждый раз, когда хотите поправить голос бренда. Голос бренда должен жить в профиле проекта.

## 2. Добавьте примеры

Примеры учат стилю лучше, чем абстрактные правила.

Для каждого формата соберите:

- 2-5 сильных примеров;
- один пример с комментариями, если возможно;
- одну пару "плохо / хорошо".

Удаляйте примеры, которые агент не должен имитировать.

## 3. Напишите редполитики форматов

У каждого формата должна быть своя редполитика.

Не пишите один огромный свод правил для всех текстов. Хорошая инструкция для статьи может быть плохой инструкцией для урока, письма или инструкции.

Используйте:

- `templates/content-type-profile.md`;
- `templates/redpolicy-template.md`.

## 4. Улучшайте после реальных запусков

Когда результат слабый, не добавляйте общие правила вроде "пиши лучше".

Добавляйте конкретную обратную связь:

- "Не начинай с определения. Начинай с ситуации читателя."
- "В этом формате не нужен призыв к действию."
- "Стоимость ставь в первый блок про инструмент, а не во вступление."
- "Расшифровывай термины до аббревиатур."

Одно точное правило лучше большого вдохновляющего абзаца.
