# Установка

Content Factory - система на Markdown-файлах. Ей не нужна сборка.

Разные агентные инструменты ожидают разные имена папок. В этом репозитории по умолчанию используются нейтральные папки:

```text
agents/
skills/
templates/
docs/
examples/
```

## Универсальное использование

Файлы можно использовать напрямую:

1. Прочитать `README.md`.
2. Заполнить шаблоны.
3. Передать нужные инструкции своему ИИ-инструменту.
4. Запустить конвейер из подходящего `skills/write-*`.

## Настройка в стиле Claude Code

Если инструмент ожидает Claude-папки, скопируйте:

```text
agents/  -> .claude/agents/
skills/  -> .claude/skills/
```

Затем оставьте `AGENTS.md` или создайте `CLAUDE.md` с теми же проектными инструкциями.

## Настройка в стиле Codex

Если инструмент читает `AGENTS.md`, оставьте корневой `AGENTS.md` как главную инструкцию.

Если настройка поддерживает пользовательские скиллы, скопируйте или свяжите:

```text
skills/ -> .agents/skills/
```

Точная папка зависит от раннера. Сама система не привязана к одному продукту.

## Первый запуск

Создайте рабочую папку:

```text
output/test-run/
```

Подготовьте:

- заполненный профиль проекта;
- заполненный профиль формата;
- редполитику одного формата;
- один исходник в `input/`;
- 2-3 примера.

Затем запустите редакционный конвейер:

```text
исследование, если нужно -> план -> проверка плана -> согласование -> черновик -> финальная редактура
```

## Что коммитить

Коммитить:

- агентов;
- скиллы;
- документы;
- шаблоны;
- нейтральные примеры;
- пустой `input/.gitkeep`;
- пустой `output/.gitkeep`.

Не коммитить:

- реальные исходники;
- сгенерированные черновики;
- финальные тексты;
- локальные настройки;
- приватные заметки;
- секреты.
